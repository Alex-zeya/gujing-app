# 股镜上线准备度

更新日期：2026-06-12

## 当前结论

股镜当前处于“可继续真机测试，暂不建议直接公开上架”的阶段。

前后端主链路已经对齐：登录、股票搜索、行情/K线、持仓、买入卖出、观察池、波动雷达和隐私入口都已经接到后端接口。内部运行监控和上线准备度保留为开发开关，不默认展示给普通用户。上线前真正卡点主要不是页面，而是生产环境、真实登录、数据源稳定性和 App Store 合规材料。

## 阶段判断

| 阶段 | 状态 | 说明 |
| --- | --- | --- |
| 本地开发可用 | 已完成 | 前端、后端、测试和 iOS sync 已跑通。 |
| 真机内测 | 进行中 | 需要继续用 iPhone 验证登录、搜索、持仓、K线、刷新和冷启动。 |
| TestFlight | 待准备 | 需要稳定 HTTPS 后端、生产数据库、公开隐私政策 URL 和 Apple/微信登录回归。 |
| App Store 上架 | 待准备 | 需要 Apple 登录能力、微信开放平台审核、数据源授权/稳定策略、截图和审核说明。 |

## 上线硬门槛

1. 后端服务
   - Render 或同类平台部署成功。
   - `/api/health` 返回正常。
   - `/api/system/readiness` 能看到数据库、行情和任务状态。

2. 生产数据库
   - 线上必须使用 PostgreSQL 或同级持久数据库。
   - 不能依赖 Web 服务本地 SQLite 文件保存正式用户数据。

3. 登录验证
   - 首版恢复手机号验证码登录，正式可用前需要接通短信服务商并确认到达率。
   - 微信登录作为国内用户增强入口，需要完成微信开放平台移动应用审核、配置 Bundle ID、Universal Links、AppID 和 AppSecret。
   - Apple 登录能力保留但当前不作为默认入口；若后续作为 App Store 主登录方式，需要在 Apple Developer 和 Xcode target 开启 Sign in with Apple。

4. 数据源
   - 免费行情源可以作为首版兜底，但要接受延迟和缺字段风险。
   - A股目录、实时/延迟行情、历史 K 线、新闻和基础字段必须有缓存、失败重试和更新时间。
   - 页面不能大量显示工程化缺失提示，缺字段时要展示为“同步中”“样本积累中”或“暂未披露”这类用户能理解的状态。

5. iOS 生产地址
   - `.env.ios` 必须指向 HTTPS 后端。
   - 不能使用 `localhost`、局域网 IP 或临时 tunnel。
   - 修改后必须执行 `npm run ios:sync`，再用 Xcode 重新安装。

6. 隐私和合规
   - 隐私政策必须有公开 HTTPS URL。
   - 所有建议类文案保持“研究辅助/风险观察”，不能承诺收益，不能写成直接买卖指令。
   - App Store Review Notes 需要解释数据来源、登录方式和免费实例冷启动。

## 下一步优先级

1. 生产环境配置审计：用开发开关查看上线准备度和真实阻塞项，普通用户界面不展示内部状态。
2. 真机回归测试：按登录、搜索、持仓、观察池、刷新、删除账户逐项跑。
3. 数据源兜底优化：统一缺字段显示规则，区分行情同步、K线样本和基本面披露状态。
4. App Store 素材：图标、启动页、截图、名称、副标题、描述、隐私政策 URL 和支持 URL。
5. 通知系统：首版可以先不做系统推送；如果要做，需要接 APNs。

```bash
npm run backend:test
npm run lint
npm run build
npm run test:e2e
npm run ios:sync
```

检查本地上线准备度：

```bash
npm run readiness
```

检查 Render 线上准备度：

```bash
npm run readiness -- https://gujing-api.onrender.com
```

## 当前上线预估

如果只做小范围 TestFlight 内测，生产后端和真机测试稳定后就可以推进。

如果要公开 App Store 上架，需要再完成 Apple 登录能力、微信开放平台审核、隐私政策公网 URL、数据源稳定策略和审核素材。按目前进度，重点不是重做产品，而是把生产配置和合规补齐。
